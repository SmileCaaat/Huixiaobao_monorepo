# 小程序（mp-weixin）改动记录

> 说明：当前仓库为 **uni-app 编译到微信小程序后的产物**（`pages/**/*.js` 等）。若你在 **源码工程** 里开发，请在对应 `.vue` 中同步以下逻辑，否则重新编译会覆盖本目录修改。

---

## 1. 维保任务列表 `pages/task/index.js`

| 改动项 | 说明 |
|--------|------|
| 查询时间范围 | 原「仅当月」`beginTime`/`endTime` 易漏掉跨月任务；改为 **`getTaskQueryTimeRange()`**：当前月往前约 12 个月首日至往后约 12 个月末日。 |
| 当前公司 | **每次** `loadTaskList` 前 **`await loadCurrentCompany()`**，避免切换公司后仍用旧 `companyId`。 |
| `taskType` 筛选 | 列表过滤改为 **`String(item.taskType ?? "") === targetType`**，兼容接口返回数字 `0`/`1` 与字符串 `"0"`/`"1"`，避免整页被滤空。 |
| `hasMore` | 按接口返回的 **`rawRows.length`** 与 `pageSize` 比较，与分页逻辑一致。 |
| `onShow` | 仅重置分页后 **`await loadTaskList()`**（不再重复单独拉公司，避免双请求）。 |

## 2. 任务详情 → 系统 → 设备链路与 `taskId`

### `pages/task/detail.js`

- 进入系统页跳转 URL 增加 **`&taskId=${taskId.value}`**，便于下游页携带任务 ID。

### `pages/task/system.js`

- 增加 **`taskId`**：`onShow` 从路由 **`options.taskId`** 或本地 **`currentTask.taskId`** 解析。
- **`loadDeviceList`** 成功后若仍无 `taskId`，尝试从 **`data.system` / `data.taskId` / 首条设备** 上的 `taskId` 回填。
- **`goDeviceDetail`**：跳转设备页 URL 增加 **`&taskId=`**。
- **消防测试 / 维护弹窗**：**`submitMaintenance`** 请求体增加 **`taskId`**；提交前校验 `taskId` 非空。
- **现场照片**：`fullImageUrl` / `pickUploadPath`；上传去掉与 `request.upload` 重复的 `showLoading`；列表展示与预览使用完整 URL；保存仍提交原始路径字符串（逗号拼接）。

### `pages/task/device.js`

- **`onShow`**：`taskId` = 路由 **`options.taskId`** 或 **`currentTask.taskId`**。
- **`loadItemList`**：若仍无 `taskId`，从 **`equipment.taskId` / 检查项首条.taskId / data.taskId`** 尝试回填。
- **`updateCheckResult`**：已带 **`taskId`**；**`handleQuickAction`** 前校验 `taskId`。
- **`updateFaultDesc`**：增加 **`taskId`**，保存前校验。
- **`updateCheckDetail`（弹窗保存详情）**：增加 **`taskId`**，**`saveDetail`** 前校验。

## 3. 维保报告列表滚动 `pages/report/index.wxss`

- 增加 **`page { height: 100%; }`**，页面根节点 **`height: 100%`**（替代仅 `100vh` 的写法，便于高度链闭合）。
- **`scroll-view` 对应 `.content`**：使用 **`flex: 1; height: 0; min-height: 0; width: 100%`**，满足小程序纵向 **`scroll-view` 必须有确定高度** 的要求，修复列表无法滑动。
- 去掉 `.content` 上重复的 **`padding-top: calc(16px + env(safe-area-inset-top))`**（导航占位已由 `uni-nav-bar` 处理），保留 **`padding: 20rpx`**。
