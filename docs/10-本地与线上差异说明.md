# 本地联调与线上差异说明

本文说明：为什么本地管理端（`http://127.0.0.1:83`）与线上（`https://huixiaobao-admin.site`）看起来不一致，以及本地测试环境如何搭建。

## 1. 结论先看

**页面代码是同一套**（同一仓库的 `backend` 模板与 Controller）。  
本地和线上不一致，主要是因为 **数据库内容不同**，不是前端/后端业务代码分叉。

| 对比项 | 线上 | 本地新建库 |
|---|---|---|
| 侧栏菜单 | 客户与合同、信息维护、消防管理等完整业务菜单 | 若依默认菜单 + 维保合同（缺公司/建筑/设备等菜单） |
| 首页统计 | 真实业务数据（公司/建筑/设备等） | 基线空库或极少样例数据 |
| 代码与模板 | `fire_main.html` 等 | 相同 |

首页数字来自接口 `/fire/stats/home` 查库；菜单来自表 `sys_menu`（及 `sys_role_menu`）。

## 2. 本地环境组成

| 组件 | 说明 |
|---|---|
| `local_sever/` | 本地联调启动台（Node 网关 + 面板按钮） |
| MySQL 8.4 | 本机 `127.0.0.1:3306`，库名 `dev_manager` |
| 后端 | `ruoyi-admin.jar`，默认端口 `83` |
| 小程序 | `miniprogram/`，由启动台切 `BASE_URL` 并打开微信开发者工具 |

一键入口：

```text
local_sever\start.bat
```

面板常用操作：

1. **Maven Clean** / **Maven Package**（进度在 `start.bat` 控制台）
2. **启动 / 打开后端界面**
3. **启动 / 打开小程序**

详细操作见 [09-local_sever.md](09-local_sever.md)。

## 3. 本地数据库怎么来的

本地库不是线上全量拷贝，而是按仓库脚本「拼」出来的：

1. `backend/sql/ry_20250416.sql` — 若依系统表与默认菜单  
2. `backend/sql/quartz.sql` — Quartz 表  
3. `backend/sql/local_fire_baseline.sql` — 本地补齐的 `fire_*` **表结构**（及极少样例）  
4. `backend/sql/add_fire_contract_module.sql` — 维保合同表与合同菜单  

仓库里 **没有** 完整的线上消防业务菜单种子（如 `fire:company:view`、客户与合同 / 信息维护等目录），也 **没有** 线上业务数据 dump。

因此：

- 本地能登录管理端、能跑通基础联调链路；  
- 侧栏与首页数字 **不会** 自动与线上一致。

合同菜单脚本依赖已存在的 `fire:company:view`；本地若没有该公司菜单，合同项可能挂在顶级，这也是菜单「看起来怪」的原因之一。

## 4. 本地 MySQL 约定

| 项 | 值 |
|---|---|
| Host / Port | `127.0.0.1:3306` |
| Database | `dev_manager` |
| User | `root` |
| Password（本机安装约定） | 见 `local_sever/config.env` 中 `DB_PASS`（该文件不提交） |
| 启动 | `local_sever\start-mysql.bat`（`start.bat` 也会尝试拉起） |

JDBC 本地需带 `allowPublicKeyRetrieval=true`（MySQL 8 `caching_sha2_password`）。`local_sever` 启动后端时已自动拼进 `DB_URL`。

> 不要把真实密码写进会提交的 `config.example.env`。本机密码只放在 gitignore 的 `config.env`。

## 5. 如何让本地「看起来像线上」

需要从 **可信生产/测试库** 导出再导入本地，至少包括：

1. **`sys_menu` + `sys_role_menu`**（以及如有依赖的字典/权限相关表）— 对齐侧栏  
2. **`fire_*` 业务数据**（公司、建筑、设备等）— 对齐首页统计与列表  

推荐在服务器上：

```bash
# 示例：仅结构+菜单与消防业务（按实际表名调整）
mysqldump -u... -p... dev_manager \
  sys_menu sys_role_menu \
  fire_company fire_building fire_equipment \
  # ... 其它 fire_* \
  > prod_fire_and_menus.sql
```

再在本机：

```bash
mysql -uroot -p... --default-character-set=utf8mb4 dev_manager < prod_fire_and_menus.sql
```

导入后重启后端，刷新 `http://127.0.0.1:83/index`。

若暂时只要菜单、不要真实客户数据：优先导入菜单与权限表即可；统计数字仍会偏少，属预期。

## 6. 其它常见差异与坑

| 现象 | 说明 |
|---|---|
| Clean 失败：删不掉 `ruoyi-admin.jar` | Windows 上 Java 进程占用 jar；启动台 Clean/Package 前会尝试停后端 |
| 启动后端超时 / Public Key Retrieval | JDBC 需 `allowPublicKeyRetrieval=true`；已在 local_sever 本地 URL 中处理 |
| 小程序连不上本地 | 确认 `BASE_URL`、开发者工具关闭域名校验、真机用局域网 IP |
| 重启电脑后 MySQL 没了 | 若已注册 `MySQL84` 服务会自动启动；否则先跑 `start-mysql.bat` 或 `Start-Service MySQL84` |
