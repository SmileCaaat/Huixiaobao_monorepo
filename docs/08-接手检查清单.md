# 接手检查清单

## 1. 需要尽快确认的信息

### 生产服务器

- 服务器地址、SSH 端口和账号。
- `/opt/app/restart-ruoyi.sh` 的完整内容。
- 当前 Java 版本和 Java 可执行文件路径。
- 实际监听端口；代码默认 83，Docker 是 80，生产脚本尚不明确。
- JAR 实际名称和进程匹配方式。
- `nohup.out` 是否有日志轮转。
- 防火墙、安全组和磁盘告警方式。

### 数据库

- 生产 MySQL 地址、版本、数据库名。
- `DB_URL/DB_USER/DB_PASS` 由脚本、systemd、shell profile 还是其他平台注入。
- 最近一次备份和恢复演练时间。
- 完整 `fire_*` 表结构以及历史迁移顺序。
- 是否允许从生产导出仅结构的脱敏基线。

### 反向代理与域名

- `huixiaobao-admin.site` 的 DNS、证书和续期方式。
- Nginx/Caddy/云托管配置位置。
- `/profile/**`、报告下载和上传是否经过代理。
- 微信公众平台配置的 request/upload/download 合法域名。

### 文件

- 生产 `uploadPath` 实际绝对路径。
- 上传文件、合同附件和报告文件的备份策略。
- 报告实际存放在 JAR 静态目录、`/opt/app/report` 还是其他位置。

### 小程序

- 微信小程序管理员和开发者权限。
- AppID、主体、体验版和正式版发布权限。
- 是否还能找到原始 uni-app 工程；如果存在，需要与当前编译产物做差异比对。
- 线上用户当前最依赖哪些页面，哪些页面可以直接废弃。

## 2. 建议首周安排

### 第 1-2 天：环境可重复

- 在本地使用 JDK 21 完成 Maven 构建。
- 获取测试数据库结构，启动后端。
- 用接口工具验证登录、公司、任务、设备和上传。
- 在微信开发者工具导入当前小程序。

### 第 3-4 天：修 P0

- 修复 401。
- 修复分页和任务筛选。
- 补 `updateMaintenance`。
- 新增正确的小程序首页统计接口。

### 第 5 天：建立基线

- 固化数据库 schema。
- 保存核心接口脱敏样例。
- 建立发布和回滚清单。
- 创建 `feature/native-rewrite` 并开始原生请求层。

## 3. 每次修改前

- 确认修改的是后端、当前编译小程序，还是未来原生分支。
- 搜索同一路径是否同时存在管理端 `/fire/**` 和小程序 `/api/fire/**` 实现。
- 核对 Java Domain、Mapper XML 和前端字段名。
- 确认是否需要数据库迁移。
- 不要在编译产物中扩大字段别名或复制请求封装。

## 4. 每次发布前

- Maven 构建成功。
- 数据库脚本经过测试环境执行。
- 旧 JAR 和数据库可回滚。
- 上传目录没有被打包或覆盖。
- 登录、公司切换、任务、设备、签到、报修、报告完成冒烟验证。
- 观察生产日志无异常后再结束发布窗口。

## 5. 当前明确缺失项

- 完整消防业务数据库初始化脚本。
- 小程序 uni-app 源工程和 sourcemap。
- 自动化测试。
- Docker Compose 或正式基础设施编排。
- 环境变量示例文件。
- 生产服务器和 Nginx 配置。
- `restart-ruoyi.sh` 源文件。
- 正式的数据库和文件备份/恢复文档。
