# 问题清单与重构路线

## 1. P0：阻断核心流程或影响安全

| 问题 | 影响 | 建议 |
|---|---|---|
| `/api/fire/task/updateMaintenance` 缺失 | 消防测试/二级维护信息无法提交 | 在小程序 REST Controller 补 JSON 接口并复用 Service |
| HTTP 401 处理错误 | Session 过期后不清 Token、不回登录 | 统一请求层同时处理 HTTP 401 和 body code 401 |
| POST 分页放在 JSON body | 多个列表可能总是默认第一页 | 明确 REST 分页 DTO 或 query 参数 |
| 任务 `taskType`/日期未在后端筛选 | 任务漏项、分页和 `hasMore` 错误 | 服务端完整应用筛选条件 |
| 首页调用 `/fire/home` | 得到 HTML，页面使用假数据兜底 | 新增公司范围的 `/api/fire/home` |
| 消防业务基线 SQL 缺失 | 无法可靠搭建新环境 | 从可信数据库导出结构并版本化 |
| 资源级权限不一致 | 可能越权访问公司、建筑、设备或巡检 | 统一 company scope 和 owner scope 校验 |
| 登录协议页面未注册 | 点击用户协议/隐私政策会失败 | 补页面或移除无效入口，满足小程序合规要求 |

## 2. P1：原生重构核心

### 阶段 1：冻结和验证

1. 从当前稳定提交创建 `feature/native-rewrite`。
2. 记录线上核心接口的脱敏请求/响应样例。
3. 建立最小回归清单：登录、公司切换、任务三级链路、设备 CRUD、签到、报修。
4. 先完成 P0 后端契约修复，避免新前端适配已知错误。

### 阶段 2：原生基础设施

优先重写：

```text
config/env.js
services/request.js
services/upload.js
services/download.js
services/auth.js
services/company.js
app.js 登录态守卫
```

要求：

- 一个请求实现。
- 一个 Token 生命周期实现。
- 一个上传/下载实现。
- dev/test/prod Base URL 分离。
- 错误码和提示可追踪，不用假数据掩盖失败。

### 阶段 3：简单页面建立原生范式

建议顺序：

1. `login/password`、`mine`。
2. `CompanyDrawer`、`index`。
3. `customer/index`。
4. `building/index/detail`。
5. `equipment/index/list/detail`。

这些页面足以建立原生导航、列表、空状态、加载态和公司上下文范式。

### 阶段 4：核心任务链

整体迁移：

```text
task/index
→ task/action
→ task/detail
→ task/system
→ task/device
```

任务链共享 `taskId`、`recordId`、`recordType` 和缓存对象，应作为一个迭代交付，避免一半使用 uni-app 状态、一半使用原生状态。

### 阶段 5：复杂表单

迁移：

- `equipment/form/edit`
- `inspection/form/detail`
- `building/form/detail`
- `repair/form/detail`
- `checkin/index`

用原生表单、picker、modal 和自建轻量组件替代 `uni-forms`、`uni-popup`、`uni-fab`。

## 3. P2：外围和清理

- 报告下载体验。
- 扫码协议标准化。
- 消息模块真实需求确认。
- 原生 tabBar 或统一自定义 tabBar。
- 删除 `common/vendor.js`、`node-modules/@dcloudio`。
- 删除无效 sourcemap 注释和字段兼容代码。
- 统一样式变量、空状态、错误页和权限提示。

## 4. 后端可独立演进项

小程序重构不要求重写后端。后端建议保持兼容式演进：

1. 把 `FireMiniAppController` 拆为多个 `@RestController`，URL 不变。
2. 为 REST API 增加 DTO，不直接把所有 Domain 当请求对象。
3. 统一分页、成功码和异常格式。
4. 建立 `CurrentCompanyService` 或统一 company scope 校验。
5. 用 Flyway/Liquibase 管理数据库版本。
6. 至少增加 Controller/Service 冒烟测试。
7. 把硬编码默认数据库地址移出代码配置。

## 5. 验收标准

### P0 完成

- Session 过期可以自动回登录。
- 任务分页和周期/临时筛选正确。
- 消防测试维护信息可提交并回显。
- 首页不再显示硬编码假统计。
- 新数据库能按文档完成初始化。

### 原生重构完成

- 不含 `common_vendor`、`createSSRApp` 和 `@dcloudio/uni-ui`。
- 31 个页面逐一确认保留、合并或删除。
- 所有线上 API 有契约记录。
- 核心流程通过真机验证。
- 微信 request/upload/download 合法域名通过检查。
- 旧编译产物在确认无回滚需要后再删除。
