# 本地更改与预览规则

本文说明：改完代码后，怎样才能在本地看到效果。面向日常用 `local_sever` 联调的开发者。

相关文档：[09-local_sever.md](09-local_sever.md)、[10-本地与线上差异说明.md](10-本地与线上差异说明.md)。

## 1. 先分清三层进程

本地联调其实是三套独立进程：

| 进程 | 入口 | 默认地址 | 改什么时要动它 |
|---|---|---|---|
| 启动台（Node） | `local_sever\start.bat` | `http://127.0.0.1:3080/` | 改 `local_sever/` 下 Node 代码或面板 |
| 后端（Java jar） | 面板「启动 / 打开后端界面」 | `http://127.0.0.1:83` | 改 `backend/` 源码或模板 |
| 小程序 | 微信开发者工具打开 `miniprogram/` | 依赖 `BASE_URL` | 改 `miniprogram/` |

**改 backend，一般不需要重开 `local_sever`。**  
启动台只负责按钮、网关和拉起 jar；它不会热加载 Java 源码。只刷新管理端页面，也看不到未重新打包的 Java 改动。

## 2. 一句话规则

| 你改了什么 | 只刷新浏览器 / 开发者工具？ | 需要 Maven？ | 需要重启什么？ |
|---|---|---|---|
| `backend` Java / Controller / Service / Mapper | 不够 | 要 **Package** | 重启后端（再点「启动 / 打开后端界面」） |
| `backend` 打进 jar 的模板、静态资源 | 不够 | 要 **Package** | 重启后端 |
| 仅数据库内容（菜单、业务表、配置） | 通常刷新即可 | 不用 | 一般不用重启 |
| `miniprogram/` 页面或 JS | 开发者工具编译/刷新即可 | 不用 | 不用重启后端；确认 `BASE_URL` 指向本地 |
| `local_sever/`（`server.js`、面板） | 不够 | 不用 | 重启 `start.bat` |

## 3. 改 backend 的标准流程

1. 修改 `backend/` 代码并保存。  
2. 打开启动台 `http://127.0.0.1:3080/`，点 **Maven Package**。  
   - 进度看 `start.bat` 那个黑窗口控制台，不要关。  
3. Package 成功后，点 **启动 / 打开后端界面**。  
   - 会用新的 `ruoyi-admin.jar` 拉起（或打开）`http://127.0.0.1:83`。  
4. 浏览器刷新管理端页面验证。

日常推荐顺序：

```text
改代码 → Maven Package → 启动/打开后端界面 → 刷新页面
```

### Clean 要不要每次做？

**不用。** 日常只 Package 即可。

出现下面情况再点 **Maven Clean**（或 Clean 后再 Package）：

- Package 失败、依赖/编译结果异常  
- Windows 上旧 jar 被占用删不掉（启动台 Clean/Package 前会尽量先停后端）  
- 大改模块，怀疑 `target/` 目录脏了  

不需要也不建议「每次改一行都 Clean + Package」。

## 4. 改小程序

1. 用微信开发者工具打开仓库下的 `miniprogram/`。  
2. 确认已切到本地：`miniprogram/utils/request.js` 里 `BASE_URL` 指向本机（如 `http://127.0.0.1:83`）。  
   - 启动台 prepare / 「启动小程序」会协助切换。  
3. 改完页面或 JS 后，在开发者工具里编译/刷新预览。  
4. **不需要** Maven，也 **不需要** 重开 `local_sever`。  
5. 若接口行为不对，再确认后端是否已用最新 jar 启动，以及本机库数据是否符合预期。

真机调试时把 `MINIPROGRAM_BASE_URL` 改成局域网 IP。上传体验版/正式版前先切回线上地址（`stop.bat` 或 prod 切换），避免把本地 URL 打进包。

## 5. 只改数据库时

例如从远程同步菜单/业务数据、改 `sys_menu`、改业务表：

- 一般 **刷新管理端页面** 即可看到菜单或列表变化。  
- 若改的是后端启动时缓存的配置，再考虑重启后端。  
- **不需要** Package，也 **不需要** 重开启动台。

本地库与线上不一致时，见 [10-本地与线上差异说明.md](10-本地与线上差异说明.md)。

## 6. 改 local_sever 本身

改了 `local_sever/server.js`、`lib/openers.js`、`public/` 等：

1. 在 `start.bat` 窗口 `Ctrl+C` 停掉。  
2. 再运行 `local_sever\start.bat`。  
3. 打开/刷新 `http://127.0.0.1:3080/`。

注意：重启启动台 **不会** 自动带上你未 Package 的 backend 改动；后端仍是当前已启动的那个 jar（若进程还在）。

## 7. 常见误区

| 误区 | 实际 |
|---|---|
| 改了 Java，只刷新 `127.0.0.1:83` | 仍是旧 jar，必须 Package 并重启后端 |
| 每次改代码都重开 `start.bat` | 改 backend/小程序通常不必；只改启动台才需要 |
| 每次都要 Maven Clean | 否，日常 Package 足够 |
| Package 完不重启后端 | jar 已更新，但旧 Java 进程还在跑旧代码 |
| 管理端菜单和线上不一致就以为代码没更新 | 多半是库数据不同，不是没 Package |

## 8. 快速对照卡

```text
改 backend 业务代码
  → Package → 启动/打开后端 → 刷新 83

改 miniprogram
  → 开发者工具刷新（确认 BASE_URL=本地）

只改 MySQL 数据
  → 刷新页面（必要时重启后端）

改 local_sever
  → 重开 start.bat → 刷新 3080
```
