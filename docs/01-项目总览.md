# 项目总览

## 1. 系统定位

汇消宝用于消防设备和维保业务管理。浏览器管理后台负责客户、建筑、设备、合同、任务、报告、权限和系统配置；微信小程序面向现场人员，负责签到、维保任务执行、设备登记、巡检和故障报修。

### 生产架构

```text
微信小程序 miniprogram/
        │ HTTPS + JSON + Bearer SessionId
        ▼
Spring Boot / RuoYi backend/
  ├─ /api/**       小程序 REST API
  ├─ /fire/**      Thymeleaf 消防管理端
  ├─ /system/**    RuoYi 系统管理
  └─ /monitor/**   监控与定时任务
        │
        ├─ MySQL
        └─ 本地文件目录 ./uploadPath
```

### 本地联调架构

```text
微信开发者工具 → miniprogram/（BASE_URL 可切到本机）
浏览器设备面板 → local_sever/（默认 :3080）
        │ 可选拉起 / 转发
        ▼
backend/（ruoyi-admin，默认 :83）
        │
        └─ MySQL（如库名 dev_manager）
```

本地一键入口见 [`local_sever/`](../local_sever/)，说明见 [09-local_sever.md](09-local_sever.md)、[10-本地与线上差异说明.md](10-本地与线上差异说明.md)。

## 2. 仓库边界

```text
Huixiaobao/
├─ backend/        Java 后端和浏览器管理后台
├─ miniprogram/    微信小程序（uni-app 编译产物，计划原生重写）
├─ local_sever/    本地测试网关、设备面板与一键启动
└─ docs/           项目文档与部署 PDF
```

### backend 的职责

- 消防业务数据模型和业务规则。
- 用户、角色、菜单、部门、权限与 Session。
- 管理端 Thymeleaf 页面。
- 小程序 REST API。
- MyBatis 数据访问、文件上传、报告、定时任务和代码生成。

### miniprogram 的职责

- 采集登录、公司、任务、设备、巡检、签到和报修信息。
- 调用后端完成实际业务读写。
- 保存后端返回的 Session ID，并作为 Bearer Token 发送。
- 不应自行承担核心业务规则；当前若干前端筛选和字段兼容逻辑应逐步下沉或收敛到明确契约。

### local_sever 的职责

- 提供本机联调网关与设备概览面板（默认 `http://127.0.0.1:3080/`）。
- 可选自动拉起 `ruoyi-admin.jar`、切换小程序 `BASE_URL`、打开微信开发者工具。
- **不是**生产部署组件；不影响线上架构。

## 3. 技术栈

| 范围 | 技术 |
|---|---|
| 后端语言 | Java；父 POM source/target 1.8，`ruoyi-system` 实际要求 Java 21 preview |
| 后端框架 | Spring Boot 2.5.15、RuoYi 4.8.2 |
| Web | Spring MVC、Thymeleaf |
| ORM | MyBatis、Mapper XML、PageHelper |
| 安全 | Apache Shiro 1.13、EhCache、数据库在线 Session |
| 数据库 | MySQL、Druid 1.2.27 |
| 工具 | Quartz、Velocity、POI/poi-tl、Swagger 3、ZXing |
| 小程序 | uni-app/Vue 编译后的微信小程序产物、uni-ui |
| 本地联调 | Node.js（`local_sever`）、PowerShell / bat 启动脚本 |
| 生产构建 | Maven 3.9 + JDK 21；仓库同时提供 Dockerfile |

## 4. 后端 Maven 模块图

```text
ruoyi-admin
├─ ruoyi-framework
│  └─ ruoyi-system
│     └─ ruoyi-common
├─ ruoyi-quartz
│  ├─ ruoyi-system
│  └─ ruoyi-common
└─ ruoyi-generator
   └─ ruoyi-common
```

| 模块 | 职责 |
|---|---|
| `ruoyi-admin` | 可执行 JAR、Controller、页面模板、静态资源和主配置 |
| `ruoyi-framework` | Spring/Shiro/Druid/MVC/异常/安全等框架层 |
| `ruoyi-system` | 系统管理和消防业务 Domain/Service/Mapper |
| `ruoyi-common` | 公共模型、注解、工具、分页、上传、Excel/Word |
| `ruoyi-quartz` | 定时任务、任务日志和调度执行 |
| `ruoyi-generator` | 数据库代码生成器 |

## 5. 入口与运行边界

### 后端入口

- 启动类：`com.ruoyi.RuoYiApplication`
- 默认端口：`83`
- 默认上下文：`/`
- Docker 运行时通过参数把端口改为 `80`

### 小程序入口

- `app.json` 的第一个页面是 `pages/login/index`。
- `app.js` 是 uni-app 编译后的 `createSSRApp` 入口，不包含有效的登录态启动守卫。
- 主业务入口为 `pages/index/index`，通过公司上下文进入任务、设备、建筑、签到、报告和报修。

### 本地联调入口

- `local_sever/start.bat`：一键启动面板与可选后端。
- 面板默认：`http://127.0.0.1:3080/`。
- 详细步骤见 [09-local_sever.md](09-local_sever.md)。

## 6. 当前成熟度判断

| 部分 | 判断 |
|---|---|
| 后端系统能力 | 较完整，沿用标准 RuoYi 架构 |
| 消防业务模型 | 较完整，覆盖公司、建筑、设备、任务、签到、巡检、报修、合同、报告 |
| 小程序 REST API | 大部分 CRUD 存在，但有若干关键契约断点 |
| 小程序代码 | 可运行的编译产物，维护成本高，缺少源工程和自动化验证；计划逐步原生重写 |
| 数据库初始化 | 仍有大量增量脚本；本地可用 `backend/sql/local_fire_baseline.sql` 作为消防模块基线建表。菜单/样例数据与线上是否一致仍取决于库内容，见 [10-本地与线上差异说明.md](10-本地与线上差异说明.md) |
| 本地联调 | 已提供 `local_sever/` 一键启动与环境切换 |
| 部署文档 | 生产侧仍主要覆盖手工更新 JAR 和查看日志（`docs/` 内 PDF） |
