# 小程序公司与任务接口对接说明

## 1. 接口范围

本文档说明小程序首页“当前公司、公司切换、我的公司列表、我的任务列表”相关接口。

接口统一前缀：

```text
/api/fire
```

涉及接口：

```text
GET  /api/fire/company/current
GET  /api/fire/company/myList
POST /api/fire/company/switch
POST /api/fire/task/myList
```

## 2. 登录态要求

这些接口都依赖当前登录用户。

小程序每次请求都必须携带后端登录后返回的会话凭证，例如 cookie、token 或项目现有 request 封装里的认证信息。

特别是：

```text
POST /api/fire/company/switch
```

会把当前公司写入后端 Session。如果后续请求没有保持同一个登录态：

```text
GET /api/fire/company/current
```

可能读取不到刚切换的公司。

## 3. 推荐调用顺序

### 3.1 首页首次加载

1. 调用 `GET /api/fire/company/current` 获取当前公司。
2. 从返回数据中取 `companyId`。
3. 调用 `POST /api/fire/task/myList` 查询当前公司的任务列表。

### 3.2 用户切换公司

1. 调用 `GET /api/fire/company/myList` 获取可切换公司列表。
2. 用户选择公司。
3. 调用 `POST /api/fire/company/switch` 保存当前公司。
4. 重新调用 `POST /api/fire/task/myList` 刷新任务列表。

## 4. 获取当前公司

### 请求

```http
GET /api/fire/company/current
```

### 说明

如果后端 Session 中已有当前公司，则返回该公司。

如果 Session 中没有当前公司，后端会自动取当前用户关联的第一个公司，并写入 Session。

当前用户关联公司的判断范围：

```text
用户是任务项目负责人 managerId
或用户是任务执行人 executorId
或用户在维保操作员 operatorIds 中
```

### 成功返回示例

```json
{
  "code": 0,
  "msg": "操作成功",
  "data": {
    "companyId": 123,
    "companyCode": "C001",
    "companyName": "某某公司",
    "projectName": "某某项目",
    "shortName": "某某",
    "companyType": "00",
    "address": "项目地址",
    "checkInLongitude": 120.123456,
    "checkInLatitude": 30.123456,
    "checkInRadius": 500,
    "checkInAddress": "打卡地址",
    "contactPerson": "联系人",
    "contactPhone": "13800000000",
    "status": "0"
  }
}
```

### 小程序取值

```js
const company = res.data
const companyId = company.companyId
```

## 5. 获取我的公司列表

### 请求

```http
GET /api/fire/company/myList
```

### 用途

用于公司切换弹窗、首页公司选择器。

### 成功返回示例

```json
{
  "code": 0,
  "msg": "操作成功",
  "data": [
    {
      "companyId": 123,
      "companyCode": "C001",
      "companyName": "某某公司",
      "projectName": "某某项目",
      "shortName": "某某",
      "companyType": "00",
      "address": "项目地址",
      "checkInLongitude": 120.123456,
      "checkInLatitude": 30.123456,
      "checkInRadius": 500,
      "checkInAddress": "打卡地址",
      "contactPerson": "联系人",
      "contactPhone": "13800000000",
      "status": "0"
    }
  ]
}
```

### 公司列表字段说明

| 字段 | 说明 |
| --- | --- |
| `companyId` | 公司 ID，后续任务查询和切换公司都用这个字段 |
| `companyName` | 公司名称 |
| `projectName` | 项目名称 |
| `shortName` | 公司简称 |
| `companyType` | 公司类型 |
| `address` | 项目地址 |
| `checkInLongitude` | 打卡中心点经度 |
| `checkInLatitude` | 打卡中心点纬度 |
| `checkInRadius` | 打卡半径，单位米 |
| `checkInAddress` | 打卡地址描述 |
| `contactPerson` | 联系人 |
| `contactPhone` | 联系电话 |
| `status` | 状态，`0` 正常 |

## 6. 切换当前公司

### 请求

```http
POST /api/fire/company/switch
Content-Type: application/json
```

### 请求体

```json
{
  "companyId": 123
}
```

### 说明

后端会校验当前用户是否有权限访问该公司。校验通过后，把 `companyId` 写入 Session，供 `GET /company/current` 读取。

### 成功返回示例

```json
{
  "code": 0,
  "msg": "切换成功",
  "data": {
    "companyId": 123,
    "companyName": "某某公司"
  }
}
```

### 失败返回示例

```json
{
  "code": 500,
  "msg": "您没有权限访问该公司"
}
```

## 7. 获取我的任务列表

### 请求

```http
POST /api/fire/task/myList
Content-Type: application/json
```

### 请求体

```json
{
  "companyId": 123,
  "taskStatus": "",
  "taskType": "0",
  "params": {
    "beginTime": "2026-04-01",
    "endTime": "2026-04-30"
  },
  "pageNum": 1,
  "pageSize": 10
}
```

### 请求参数说明

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| `companyId` | 建议传 | 当前公司 ID |
| `taskStatus` | 否 | 任务状态，空字符串表示全部 |
| `taskType` | 否 | 任务类型，`0` 周期任务，`1` 临时任务，空字符串表示全部 |
| `params.beginTime` | 否 | 开始日期，格式 `yyyy-MM-dd` |
| `params.endTime` | 否 | 结束日期，格式 `yyyy-MM-dd` |
| `pageNum` | 否 | 页码，默认第 1 页 |
| `pageSize` | 否 | 每页数量，默认 10 |

时间范围按任务的计划开始时间 `planStartTime` 过滤。

### 权限说明

小程序不要传 `userId`。

后端始终使用当前登录用户 ID，只返回当前用户相关的任务。

相关任务判断范围：

```text
用户是任务项目负责人 managerId
或用户是任务执行人 executorId
或用户在维保操作员 operatorIds 中
```

### 成功返回示例

```json
{
  "code": 0,
  "msg": null,
  "total": 25,
  "rows": [
    {
      "taskId": 1,
      "taskName": "4月维保任务",
      "companyId": 123,
      "companyName": "某某公司",
      "planStartTime": "2026-04-01 00:00:00",
      "planEndTime": "2026-04-30 23:59:59",
      "taskStatus": "0",
      "totalItems": 100,
      "completedItems": 20,
      "normalItems": 18,
      "faultItems": 1,
      "noDeviceItems": 1,
      "executorId": 12,
      "executorName": "张三",
      "taskType": "0",
      "managerId": 8,
      "managerName": "李四",
      "operatorIds": "12,15",
      "operatorNames": "张三,王五",
      "buildingId": 10,
      "buildingName": "1号楼"
    }
  ]
}
```

### 返回字段取值

```js
const list = res.rows
const total = res.total
```

### 任务人员字段说明

| 字段 | 说明 |
| --- | --- |
| `managerId` | 项目负责人 ID |
| `managerName` | 项目负责人姓名 |
| `executorId` | 执行人 ID |
| `executorName` | 执行人姓名 |
| `operatorIds` | 维保操作员 ID，多个用英文逗号分隔 |
| `operatorNames` | 维保操作员姓名，多个用英文逗号分隔 |

## 8. 小程序请求示例

### 8.1 首页加载当前公司任务

```js
async function loadHomeTasks() {
  const currentRes = await request({
    url: '/api/fire/company/current',
    method: 'GET'
  })

  const companyId = currentRes.data.companyId

  const taskRes = await request({
    url: '/api/fire/task/myList',
    method: 'POST',
    data: {
      companyId,
      taskStatus: '',
      taskType: '',
      params: {
        beginTime: '2026-04-01',
        endTime: '2026-04-30'
      },
      pageNum: 1,
      pageSize: 10
    }
  })

  return {
    company: currentRes.data,
    tasks: taskRes.rows,
    total: taskRes.total
  }
}
```

### 8.2 切换公司后刷新任务

```js
async function switchCompany(companyId) {
  await request({
    url: '/api/fire/company/switch',
    method: 'POST',
    data: {
      companyId
    }
  })

  return request({
    url: '/api/fire/task/myList',
    method: 'POST',
    data: {
      companyId,
      taskStatus: '',
      taskType: '',
      params: {},
      pageNum: 1,
      pageSize: 10
    }
  })
}
```

## 9. 常见问题

### 9.1 `current` 接口提示未找到关联公司

说明当前登录用户没有任何关联任务公司。

后端关联公司来自任务表，满足以下任意一个条件才算关联：

```text
managerId = 当前用户
executorId = 当前用户
operatorIds 包含当前用户
```

### 9.2 切换公司成功后，重新进入又变回其他公司

优先检查小程序请求是否保持同一个登录态。

`switch` 保存的是后端 Session，后续请求如果没有携带同一会话，`current` 就无法读取上一次切换结果。

### 9.3 任务列表为什么不用传 `executorId`

`/task/myList` 是“我的任务”接口，后端会自动使用当前登录用户 ID 做权限过滤。

前端传 `executorId` 没有必要，也不应该用它来控制能看谁的任务。

### 9.4 `taskStatus` 和 `taskType` 怎么查全部

传空字符串或不传都表示全部：

```json
{
  "taskStatus": "",
  "taskType": ""
}
```
